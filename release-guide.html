Release Guide

Prerequisites

You need to edit your ~/.m2/settings.xml to add the camel-extra servers. So your XML should look something like...

<?xml version="1.0"?>
<settings>
    <servers>
        <server>
            <id>sonatype-nexus-staging</id>
            <username>MY_SONATYPE_USERNAME</username>
            <password>MY_SONATYPE_PASSWORD</password>
        </server>
        <server>
            <id>sonatype-nexus-snapshots</id>
            <username>MY_SONATYPE_USERNAME</username>
            <password>MY_SONATYPE_PASSWORD</password>
        </server>
...

If you are a committer you should then be able to submit a snapshot via

mvn deploy

Before preparing the release

  • make sure that you can build project locally for branch being released with the following command mvn clean install -PwithRatCheck
  • make sure that build for branch being released succeeds on the CI server

Performing a release

Use the normal maven release plugin.

mvn release:clean
mvn -Dusername=MY_GOOGLE_ACCOUNT_NAME -Darguments="-Dgpg.passphrase=MY_GPG_PASSPHRASE" release:prepare
mvn -Dusername=MY_GOOGLE_ACCOUNT_NAME -Darguments="-Dgpg.passphrase=MY_GPG_PASSPHRASE" release:perform

Activating Maven Central synchronization

  • close release candidate staged to the Sonatype Nexus
  • send staging repository link to the mailing lists so the others could verify the release candidate
  • collect +1s from the other Camel Extra riders :)
  • release staged artifacts via Sonatype Nexus

More information related to the Sonatype release process can be found in the Sonatype OSS Maven Repository Usage Guide.
