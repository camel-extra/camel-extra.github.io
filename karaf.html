Karaf Support

Like the Apache Camel project, Camel-Extra is designed to work within the Apache Karaf OSGi container.

As of Camel-Extra 2.11.0 we provide feature descriptors allowing to easily install any camel-extra component and resolve their dependencies.

Preparing Karaf for Camel-Extra

The preparation instructions for Karaf correspond the preparation section of the Camel Karaf documentation.

NOTE: Use of camel-db4o Component

In addition to the standard preparations the camel-db4o component requires the setup of another maven repository to resolve the 'com.db4o/db4o-core-java5/8.0' bundle. One option is to add the following line to your 'org.ops4j.pax.url.mvn.cfg' in your '$KARAF_HOME/etc/' folder to your 'org.ops4j.pax.url.mvn.repositories':
http://svn.codespot.com/a/apache-extras.org/camel-extra/maven2/others/@id=camel-extra-others

Install Camel-Extra in Karaf

To add the camel-extra features, you need to register the feature descriptor via the following command:

karaf@root> features:addurl mvn:org.apache-extras.camel-extra.karaf/camel-extra/2.11.0/xml/features

Now you can find all camel-extra features that are available:

karaf@root> features:list | grep camel-extra

[uninstalled] [2.11.0 ] camel-exist camel-extra-2.11.0

[uninstalled] [2.11.0 ] camel-jgroups camel-extra-2.11.0

To install a component, you can execute:

karaf@root> features:install camel-rcode
karaf@root> features:list | grep camel-rcode
[installed ] [2.11.0 ] camel-rcode camel-extra-2.11.0

All dependencies that are required for the component will be pulled automatically. In the given example , the feature resolves the following dependencies:
